# Prompt Optimization Skill

사용자의 프롬프트를 체계적으로 분석하고 개선하는 전문 스킬입니다.

## 활성화 조건

이 스킬은 다음 상황에서 자동으로 활용됩니다:
- 사용자가 모호하거나 불완전한 프롬프트를 입력할 때
- 프롬프트 개선이나 최적화를 요청할 때
- "프롬프트", "질문 개선", "더 구체적으로" 등의 키워드가 포함될 때

## 평가 프레임워크

### 품질 평가 기준 (0-100점)

```
총점 = 명확성(25) + 맥락(25) + 목표(25) + 제약조건(25)
```

| 점수 구간 | 품질 수준 | 권장 조치 |
|----------|----------|----------|
| 0-39점 | 매우 부족 | 기본 정보 추가 필요 |
| 40-59점 | 부족 | 직접 입력으로 보충 |
| 60-74점 | 보통 | 객관식으로 구체화 |
| 75-89점 | 양호 | 미세 조정 |
| 90-100점 | 우수 | 최적화 완료 |

### 평가 상세 기준

**명확성 (Clarity)**
- 요청의 핵심이 명확한가?
- 모호한 표현이 없는가?
- 해석의 여지가 최소화되었는가?

**맥락 (Context)**
- 배경 정보가 충분한가?
- 상황 설명이 적절한가?
- 필요한 전제 조건이 명시되었는가?

**목표 (Goal)**
- 원하는 결과물이 분명한가?
- 성공 기준이 정의되었는가?
- 기대하는 형식이 명시되었는가?

**제약조건 (Constraints)**
- 제한사항이 명시되었는가?
- 범위가 적절히 정의되었는가?
- 예외 사항이 고려되었는가?

## 개선 전략

### 직접 입력 모드 (< 60점)

프롬프트가 기본적인 정보가 부족할 때 사용합니다:

1. **부족한 요소 식별**
   - 누락된 평가 기준 파악
   - 가장 시급한 개선 포인트 도출

2. **구체적 안내 제공**
   - 추가해야 할 정보 명시
   - 예시와 함께 가이드라인 제공

3. **사용자 입력 통합**
   - 기존 프롬프트와 새 정보 병합
   - 자연스러운 문장으로 재구성

### 객관식 모드 (60-89점)

프롬프트가 기본 형태를 갖췄지만 구체화가 필요할 때:

1. **핵심 질문 도출**
   - 프롬프트 개선에 가장 효과적인 질문 선정
   - 한 번에 하나의 측면만 질문

2. **선택지 설계**
   - 4가지 상호 배타적 옵션
   - 각 옵션이 구체적이고 실용적
   - "기타" 옵션으로 유연성 확보

3. **반복적 개선**
   - 최대 5회까지 질문-선택-개선 반복
   - 매 반복마다 점수 재평가
   - 90점 이상 시 조기 종료

## 종료 조건

다음 중 하나를 만족하면 최적화를 종료합니다:
- **점수 달성**: 90점 이상
- **반복 완료**: 객관식 질문 5회 완료

## 출력 형식

### 평가 결과
```
📊 현재 점수: {score}/100
📋 모드: {직접 입력 | 객관식}
```

### 최종 결과
```
✨ 최적화 완료!
📊 최종 점수: {score}/100
🔄 질문 횟수: {count}회

🎯 최종 프롬프트:
{optimized_prompt}
```
